delete/1

delete(Channel) -> any()

Return permissions for the given channel. The module implementing supercast_channel behaviour must return the #perm_conf{} defining the authorisation to subscribe to him. Triggered on a client call to sync_request, supercast define if the user is allowed and continue or stop subscription process.

subscribe/2

subscribe(Channel::string(), CState::#client_state{}) -> ok

Called by a channel to subscribe a client to himself. Every following emit/2 messages will then be delivered to the client. This function must be called in the sync_request/2 callback to effectively subscribe the client.

unicast/2

unicast(Client::#client_state{}, Msg::[Msg::{function, function()} | {pdu, tuple()}]) -> ok

Send a message to an unique client. Used by a channel to send a list of message or funs to a single client identified by CState wich is a #client_state. fun() will just be executed as is in the client loop with #client_state{} as argument. You will need tu use it (specialy #client_state.socket and #client_state.encoding_mod) to encode and send messages. You must use the client_state to send messages when building the fun.
